问题描述

一个搭在自己服务器的网站拒绝服务

这个服务器关过机
oa系统
刚装上的时候都可以
现在我们要进到系统里取证，拒绝访问了
我们怀疑过域名是否过期，确认没问题

服务器的管理平台可以登录，无法远程连接，所以我们把服务器当做主机，连接了显示屏，是 ubuntu 的命令行
一共三台，对方也没提供明确的账号密码及对应关系。我们昨天晚上最后登进去了

详情推测

集群情况：总共有三台服务器，三台服务器能够进入其系统，皆为 ubuntu 系统。

可用软件平台：
1. 服务器管理平台，由三台服务器共同搭建；[能够登入，无法远程连接]
2. oa系统；[使用域名访问，无法连接，拒绝服务]

目标，登入 oa 系统。

可能原因：

  1. 网站负载的 serve 未启动；
  2. 防火墙；[小概率]
  3. 域名过期;[已排除]

解决方法

1. 查验网站的服务是由什么构建的

常用的 web 服务有 nginx 、 apache 、 tomcat。

首先要确定其服务， 先广泛检索一下有没有 web 服务：
ps -ef|grep -E 'nginx|Apache|Tomcat|httpd'

其中 httpd 是 apache 的服务。

此时应该能够确定，如果没有可用显示，就检索一下文件，根目录(root)下：

  1. find / -name nginx
  2. find / -name Apache
  3. find / -name Tomcat
  4. find / -name httpd

如果系统中有它们的文件，那么应该就是用的这个服务。

2. 启动服务

看看其状态，常用 web 服务即其查看服务命令(systemctl 系)：

  1. Nginx：
    1. systemctl status nginx.service
  2. Apache：
    1. systemctl status httpd
  3. Tomcat
    1. ps -ef|grep tomcat

tomcat 是轻量级的，服务启动与其他不同，参考Linux查看tomcat启动状态，查看tomcat日志

如果是 Nginx 或 Apache， 直接用 systemctl 或者 server 命令启动就好。

tips

一般启动后就能够访问了。
如果不行，那再看，可能是端口占用或者其他问题也说不定。

其他建议

  1. 直接使用 ip:port 方式访问，不使用域名；

附录

常用 web 服务即其查看服务命令(systemctl 系)：

  1. Nginx：
    1. systemctl status nginx.service
  2. Apache：
    1. systemctl status httpd
  3. Tomcat
    1. ps -ef|grep tomcat

广泛检索服务和文件：

  1. 查验正在运行的所有中是否有常用的 web 服务：
    ps -ef|grep -E 'nginx|Apache|Tomcat|httpd'
  2. 找文件，根目录(root)下：
    1. find / -name nginx
    2. find / -name Apache
    3. find / -name Tomcat
    4. find / -name httpd
