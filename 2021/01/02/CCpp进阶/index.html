# C/C++ 进阶

为什么要进阶？
计算机到底是在学什么呢？一个是教程性质的，一个是思想性质的。
不要忘记语言只是工具，效率是追求，算法是工具的基本，数学思想才是根本。

先说 C 语言，以下都是基于 C 的，其中问题索引包含 C 语言，之后会专门开一阙记录 C 语言指针。
整个文章自上而下。

# 基本思想

C++ 是面向对象的程序设计方法。
定义？其实 OPP (Object Oriented Programming) 就是一种模块化的思想。
专业的定义是将现实的东西进行抽象化描述为对象并建立模型。本质上是分类并集合，塑造并构建。

我们可以大致想象一下编程的过程，写很大 (取极限) 的程序的时候，头文件一堆，函数也是一堆的，只学了 C 语言的基础的时候，用一个数组在描述海量的东西，二维数组就在存放着各种自定义的数据，它就可以被解释为一个对象，联合头文件中所存有的方法，就可以认为是一种 class。
基于此，然后就诞生了衍生出更加快捷，即便于理解，又符合逻辑效率的，面向对象的编程方法。
话不多说直接复习。

# 继承与派生

软件重用性的体现。

一个新类冲已有的类那里获得其已有的特性，这种现象称为类的继承 (inheritance)；从一个已有的类 (父类) 中产生一个新的子类，称为类的派生。

派生类的基类的具体化，基类是派生类的抽象。

class 派生类名: [继承方式] 基类名
{
派生类新增加的成员；
}

继承方式包括：

  1. public：基类的公有成员和保护成员在派生类中保持原有访问属性，其私有成员仍为基类私有。
  2. private：基类的公有成员和保护成员在派生类中成了私有成员，其私有成员仍为私有。
  3. protected：基类的公有成员和保护成员在派生类中成了保护成员，私有成员仍未基类私有。

保护成员的意思是，不能被外界引用，但可以被派生类的成员引用。

  • 派生与析构函数

    派生类中执行析构函数时，一般先调用派生类析构函数再调用基类析构函数。
    但如果是用 new 运算符建立临时对象，并定义了一个指向基类的指针变量。在程序中用带指针参数的 delete 撤销对象时，系统只会执行基类的析构函数。
    如果要执行派生类的析构函数，需要声明基类析构函数为虚函数。

      Point *p = new Circle;

# 多态与虚函数

多态就是不同的集合对相同的指令做出不同的回应。
专业术语：向不同的对象发送同一个消息，不同的对象再接收时会产生不同的行为 (即方法)。
C++ 的虚函数就是用来解决动态多态问题的。

在类中声明虚函数。其派生类中的同名函数都自动称为虚函数，派生类中声明可以加也可以不加 virtual，但是为了方便辨识统一加上。

virtual void display();

虚函数原理基于：

  1. 我们可以通过父类定义的指针去指向子类。
  2. 派生类重新定义父类同名函数，仍然执行父类同名函数。

所以虚函数类似于 JAVA 中的重写功能。

# 纯虚函数与抽象类

纯虚函数：

virtual void shapeName() const = 0;

只有虚函数的类，为多态性服务的类称为抽象类。

# 指针

为什么要一上来就放指针？因为指针和 OOP 编程思想才是 C++ 的关键，其他的一概其次。

指针是基于存储数据时有一个地址。
地址指向该变量单元。因此将地址形象化为指针。通过指针就可访问以它为地址的内存单元。一个变量的地址称为该变量的指针。
一个变量准们用来存放地址 (即指针) 的，则称它为指针变量。

定义

int *i_pointer;

用 int 定义的指针变量只能存放 int 类型的数据的地址。

int i;
i_pointer = &i;

(1) &*pointer_1 表示：指针 pointer_1 指向的变量的地址。
(2) *&a 表示：等价于变量 a。

& 为取地址运算符，将之后的变量地址取出；而 * 则是之后变量 (解释为地址) 所指向的变量。

用指针作为函数参数，在函数内改变指针指向的变量的值，那么这里面的值将会改变。
换而言之，使用 * pointer 时会改变其数值，但是使用 pointer 时不会改变，因为其作为形参的本质没有改变。

指针在指向数组或者字符串的时候，通常可以使用 p+i 的方式进行对数组或者字符串中的数据进行指向。

const int *p; const 指针，不能通过指针 p 来改变指针所指向的变量的值，但是可以改变指针的指向。
char * const p; 表示指针指向的变量是不能改变的。
const int * const p; 表示指向不能改变并且指向的变量值也不能改变。

void *p; 指针 p 指向空类型或不指向确定的类型，简而言之就是不能指向任何变量，如果要指向变量必须通过类型强制转换 p1 = (char\*)p2 。

# 引用

引用声明符和取地址符都是 &。引用声明符只出现在数据类型名后面。

int &b = a;

引用在声明时必须使其初始化。不能引用数组，不能建立引用的引用。不能建立指向引用的指针。

引用在调用函数时使用，可以将参数进行’变量的地址传递’，所以这种传递是可以改变数据的。是一种传址方式。
而传递变量和传递指针，本质上都是传值方式。

形参 parameter 是在定义函数名和函数体的时候使用的参数，目的是用来接收调用该函数时传入的参数
实参 argument：在调用有参函数时，主调函数和被调函数之间有数据传递关系。在主调函数中调用一个函数时，函数名后面括号中的参数称为 “实际参数”（简称 “实参”）。

# 数据的存储

内存中供用户使用的存储空间分 3 部分：

  1. 程序区
  2. 静态存储区
  3. 动态存储区

存储期 storage duration，分为静态存储期和动态存储期。
存储方法分为静态存储和动态存储两大类。
同时存储类别也分为 4 种：自动的 (auto)、静态的 (static)、寄存器的 (register)、外部的 (extern)。

Auto：
自动分配内存。属于动态局部变量，函数调用结束后即释放。编译后如果没有赋值是一个不确定的值。

Static：
指定为静态局部变量。在静态存储区内分配存储单元。在整个运行期间都不释放。编译后如果没有赋值是 0 或者空字符。
每次调用该函数，该变量保存上一次使用的值。

Register：
Auto 和 Static 都是存放在内存中的，使用的时候由控制器发出指令进行调用到运算器。对于使用频繁的变量使用 Register 放在寄存器中，可以提高执行效率。

Extern：
声明全局变量。与 Static 的区别是，static 是只能用域本文件的外部变量 (全局变量) 称为静态外部变量。。

而 Extern 和 Static 也可以用于函数。分外部函数和内部函数。

# 数值型和字符型数据的字节数和数值范围

数据如果指定时 unsigned，那么将会以补码的方式进行存放。最高位 bit 表示数值符号。二进制存放。
但是无符号型和有符号型除了表示的范围不同，那么他们是否表达相同的长度呢？其实是一样的，

  1. 就逻辑而言，因为是二进制，所以每多一位则翻一倍即 * 2，有符号型 7 位 * 2 有正有负 + 0，无符号型 8 位，是一样的。
  2. 就数理而言，8 个单元，每个单元两种可能，可以取到的情况数是一样的。

# 标准输入输出流的控制符

dec 十进制输出 (设置数值的基数为 10)
hex 十六进制
oct 八进制
