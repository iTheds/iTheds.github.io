# B 树学习札记

B 树是一种平衡多叉树，适用于外查找。
每个节点可以存储多个关键字，即多个数据。

m 阶 B 树 (Balanced tree of order m) 是一棵平衡的 m 路搜索树。它或者是空树，或者是满足下列性质的树：

  1. 根节点至少有两个子女；
  2. 每个非根节点包含的关键字个数 j 满足：┍m2┑−1≤j≤m−1┍\frac{m}{2}┑-1 ≤ j ≤ m -1┍2m​┑−1≤j≤m−1
  3. 除根节点和叶子节点外，所有节点的度数，正好是关键字总数 +1 ，故内部子树个数 k 满足：┍m2┑≤k≤m┍\frac{m}{2}┑ ≤ k ≤ m┍2m​┑≤k≤m;
  4. 所有的叶子节点都位于同一层。

阶数：B 树中的特有，与度数同。
度数：最大子节点数目。
关键字：即所含有的数据量。一般的二叉树，一个节点只有一个数据，但是 B 树中可以有多个数据，即多个关键字。

B 树中，节点中的关键字升序排列 ASC，如果子节点中有非叶子节点，则 k-1 关键字刚好是 k 个孩子节点包含的关键字的值域划分。
因为叶子结点不包含关键字，所以可以把叶子结点看成在树里实际上并不存在外部结点，指向这些外部结点的指针为空，叶子结点的数目正好等于树中所包含的关键字总个数加 1。

B - 树中的一个包含 n 个关键字，n+1 个指针的结点的一般形式为： (n,P0,K1,P1,K2,P2,…,Kn,Pn)
其中，Ki 为关键字， K1<K2<…<Kn , Pi 是指向包括 Ki 到 Ki+1 之间的关键字的子树的指针。

# B 树插入

插入的时候，我们需要记住一个规则：判断当前结点 key 的个数是否小于等于 m-1，如果满足，直接插入即可，如果不满足，将节点的中间的 key 将这个节点分为左右两部分，中间的节点放到父节点中即可。

如果这个树满了呢。

# B 树删除

删除也是根据删除后 key 的个数是否大于等于 m/2，如果满足，直接删除；不满足，则要先向兄弟节点借数据，向一个方向移动，一个兄弟节点成为父节点，一个父节点称为自己的的关键字。

如果兄弟节点也不够了，那么则合并。

但是在 B 树的插入和删除中可能会导致某一个子树高度超标，这个时候怎么办呢？分裂或者合并高层？

# B 树的运算准则

B 树主要是为了解决二叉树中 (log2nlog 2 nlog2n) 在数量庞大时，因为深度的扩展而效率接近二分法的问题。
使用多关键字、多叉的方式提高搜索效率。
B + 树还有一个最大的好处，方便扫库，B 树必须用中序遍历的方法按序扫库，而 B + 树直接从叶子结点挨个扫一遍就完了，B + 树支持 range-query 非常方便，而 B 树不支持。这是数据库选用 B + 树的最主要原因。

从平衡二叉树、B 树、B + 树、B * 树总体来看它们的贯彻的思想是相同的，都是采用二分法和数据平衡策略来提升查找数据的速度；
不同点是他们一个一个在演变的过程中通过 IO 从磁盘读取数据的原理进行一步步的演变，每一次演变都是为了让节点的空间更合理的运用起来，从而使树的层级减少达到快速查找数据的目的；
