# 630科考船数据项目

该项目起始于2021年4月6日

关于整改并编写船舶数据项目的建议：
1. 目前项目由三个exe程序组成，太过于冗余繁杂。并且主程序和监控程序之间通过输出管道进行通信，该方式风险更大。故而建议将两个程序合并成一个。或者将监控程序的功能单一化。
2. 项目增加 QT 界面并以应用软件而非控制台的方式存在。
3. 将网络体系使用更可复用的代码体现。使用 数据库 server 中的网络组件。
4. 更新为 mco 层接口。
5. 更细化表的结构。
6. 我希望程序可以更加智能的进行执行，拥有更好的易扩展性。
7. 保证健壮性应该通过完善程序的总体设计和异常处理，再加上程序对线程的数据分离以及数据控制。而不是去再编写一个程序监控另一个程序的运行状态，这样只是徒增工作量，同时也加大了风险。
8. 远程更新完善。

具体细节优化与设计：
1. 增加每个类的输出方式。

## 主要需求内容

1. 船载数据库方，接收 8 种原始数据源，并且转发响应尽可能多的用户。
2. 针对原始数据进行解析，并且存储到数据库，实现持久化存储。
3. 兼具 QT 界面方便使用。
4. 支持局域网内远程访问服务，满足最基本的增删改查。供融合子系统进行查询获取 8 种存储好的数据。
5. 确保运行的健壮性，稳定运行。
6. 数据最快每秒发送 5 次。
7. 据分析，每月数据量至少为 200 MB。

## 详细设计

一个端口就可以支持对多个客户端的数据发送。所以船舶项目的端口数可以从 8*8 锐减到 1 。但是前提就是约定了一个通信协议，来指定发送何种数据。
次级选择就是，开启与数据数量同等量的端口，只要连接，就开始发送数据，所以只需要 8 个端口。
所以现在就分为两个部分，数据采集，数据订阅转发。

我们可以这样，设计一个只使用单端口进行通信的组件，该组件是可以复用的，直接复用到 远程访问服务 组件。
然后在本项目中，开辟若干个线程去响应。节省工作量。

程序过程：

读取采集信息表<标识了需要采集的数据项以及其 ip、地址等>，通过配置文件读取。

初始化网络：
分配相应量的线程<A, n>()进行收集数据。
分配监听连接的线程。

线程<A, n>()：
请求向上连接
建立连接
分配线程<B, 1>()进行专项接收数据。
该部分是否需要再开辟线程解析存储数据？如果开辟则需要进行线程间切换。
如果不需要则直接在该线程中完成此步骤。

线程<B, 1>()：
专门接收数据的线程。
是否有必要分配线程以接收数据呢。感觉必要性不是很大，因为socket本身就维护了一个缓冲区。

如果不需要<B, 1>的话，那么流程为：
1. 分配 n 个线程接收 n 个端口的数据。
2. 对数据进行存储。
3. 分配 n 个线程进行监听连接，生成相应数目的连接队列。
4. 根据线程中的连接队列，发送数据。

本次重构先把现有的功能整理，然后进行替换。

我们希望向上嗅探和向下分发两者分离开来。
之前的是通过结构体 SOCKET_INFO 将两者存放在一起。

开发项目的受应该先考虑数据的流和数据处理，之后再考虑功能性。

我居然以为我会面向对象的程序设计，其实我只会面向过程的。
确定数据，编写功能函数，将函数添加到类内。

原本的程序中通过 recv 判断连接是否断开，如果断开则进行重连。
比较好的做法是，如果连接断开，则将数据发送并存储完成。

看来写程序之前画好流程图是非常关键的一件事。

分配一个子线程，子线程通过配置表进行相关操作。

我知道干着事需要多花时间，但是还是坚持弄好叭。
其实整个系统的代码数量，也不过就是 1500 行而已。

[7.6]出差时间日志
首先目前的问题是，更新完成之后出现了数据粘连的情况。所有数据会发送到新的连接里面。
初始化时，给了每个线程一个发送的 send_list。并且初始化到监听的网络中。
如果监听到新的连接，就将新的连接添加到对应的 send_list ，当 send() 的时候将数据发送到目标位置。
这个过程是分离性质的，不太可能是 buffer 粘连。
极有可能的就是， send 的时候，监听的新连接绑定的位置问题，新的连接绑定到的，或者说这些连接都是一起的 send_list ，所以绑定之后，整个数据变得混乱，每个数据源接受到数据后，都向同一个连接进行发送，而其他的地方却没有再进行发送。

分析得到，程序将数据发向同一个订阅数据连接。
确定是监听连接时出现的问题。
已经找到问题，因为监听连接的时候，new_connect 没有进行重新分配，所以该指针一直在被使用。导致其他的分发内容使用时，调用的都是相同的指针内容，将数据发送到一起。
在更新通讯代码之前，无论是什么情况都会重新赋值，所以没有出现该情况。

所以，如果要构造一个给全局改变的变量，可以通过将变量寄存到一个cpp 文件中，然后在头文件中声明为外部变量。
