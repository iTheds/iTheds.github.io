# 网络安全扫描技术

## 实验原理

黑客攻击通常分为三个阶段：信息搜集、攻击实施、隐身巩固。在信息搜集阶段中，扫描器发挥着至关重要的作用。根据扫描技术的不同，扫描器可以分为以下几类：

### 1. 主机扫描技术

主机扫描的目的是确定目标网络上的主机是否可达。这是信息收集的初级阶段，其效果直接影响到后续的扫描工作。主机扫描通常通过发送ICMP请求或TCP/UDP探测包来实现。

### 2. 端口扫描技术

确定目标主机可达后，使用端口扫描技术发现目标主机的开放端口，包括网络协议和各种应用监听的端口。端口扫描是最常用的扫描手段，可以揭示目标系统提供的网络服务。

### 3. 栈指纹OS识别技术

根据各个操作系统在TCP/IP协议栈实现上的不同特点，采用黑盒测试方法，通过研究其对各种探测的响应形成识别指纹，进而识别目标主机运行的操作系统类型和版本。

### 4. 漏洞扫描技术

在端口扫描后得知目标主机开启的端口以及端口上的网络服务，将这些相关信息与网络漏洞扫描系统提供的漏洞库进行匹配，查看是否有满足匹配条件的漏洞存在。通过模拟黑客的攻击手法，对目标主机系统进行攻击性的安全漏洞扫描，如测试弱口令等。若模拟攻击成功，则表明目标主机系统存在安全漏洞。

## 端口分类

通常人们将端口分为如下两类：

- **公认端口**：从0到1023，这些端口由IANA分配。有时，相同端口号就分配给TCP和UDP的同一服务。这些端口监听守护进程往往是约定俗成的一些服务，通常只有系统进程可以使用它们；一些入侵者利用了这些端口，则潜在地控制了整个系统。

- **已注册端口**：从1024到65535，这些端口不受IANA控制，但由IANA登记并提供其使用情况清单，以方便整个群体，并且不要求在此监听的守护进程具有管理员权限。

## 端口扫描技术详解

端口扫描是向目标主机的TCP/IP服务端口发送探测数据包，并记录目标主机的响应的技术。通过分析响应来判断服务端口是打开还是关闭，就可以得知端口提供的服务或信息。

按照端口连接的情况，端口扫描可分为以下几类：

### 1. TCP connect()扫描

TCP连接扫描是向目标端口发送SYN报文，等待目标端口发送SYN/ACK报文，收到后向目标端口发送ACK报文，即著名的"三次握手"过程。在许多系统中只需调用connect()函数即可完成。

**优点**：
- 不需要特殊权限，系统中的任何用户都有权使用
- 通过同时打开多个套接字可以加速扫描

**缺点**：
- 会在目标主机的日志记录中留下痕迹，易被发现
- 数据包可能会被防火墙过滤掉
- 目标主机的logs文件会显示一连串的连接和连接出错的服务信息

### 2. TCP SYN扫描（半开放扫描）

TCP SYN扫描利用TCP三次握手的特点来实现。通过向远程主机的端口发送一个SYN数据报文：
- 如果收到RST数据报文，说明端口关闭
- 如果收到SYN/ACK应答，说明端口开放

在收到SYN/ACK后，扫描器不再发送ACK应答，因此三次握手未完成，正常的TCP连接无法建立。

**优点**：
- 扫描信息不会被记入系统日志
- 在目标主机上留下的记录较少

**缺点**：
- 现代防火墙和IDS系统已经能够检测这种扫描
- 需要较高的系统权限来构造原始数据包

### 3. 秘密扫描

秘密扫描能躲避IDS、防火墙、包过滤器和日志审计，从而获取目标端口状态信息。由于没有TCP三次握手的任何部分，所以更难被记录下来，比半连接扫描更为隐蔽。

**缺点**：
- 扫描结果的可靠性较低
- 扫描主机需要构建自己的IP包

#### 3.1 TCP FIN扫描

许多过滤设备能过滤SYN数据报文，但允许FIN数据报文通过。FIN扫描的原理是：
- 向目标主机发送FIN数据包
- 如果是开放端口，数据包会被丢弃，不返回任何信息
- 如果是关闭端口，会返回RST数据包

这种方法与系统的TCP/IP实现有关，不适用于所有系统。例如，某些系统无论端口是否打开，都会回复RST，但这种方法可以用来区分UNIX和Windows系统。

#### 3.2 TCP ACK扫描

主要用于探测防火墙的过滤规则：
- 发送ACK报文
- 如果端口被防火墙过滤，要么收不到报文，要么收到ICMP(目标不可达)
- 如果端口没有被过滤，则收到RST报文

#### 3.3 NULL扫描

将TCP数据包中的所有标志位（ACK、FIN、RST、SYN、URG、PSH）置空后发送：
- 如果端口开放，目标主机不返回任何信息
- 如果端口关闭，目标主机返回RST信息

#### 3.4 XMAS扫描

将TCP数据包中的所有标志位（ACK、FIN、RST、SYN、URG、PSH）置1后发送：
- 如果端口开放，目标主机不返回任何信息
- 如果端口关闭，目标主机返回RST信息

需要注意的是，某些操作系统（如Windows、Cisco、BSDI、HP/UX、MVS、IRIX）对于FIN、XMAS和NULL扫描，即使端口开放也会发送RST数据包。这种情况下，可以通过TCP SYN扫描来确定操作系统类型。

#### 3.5 SYN/ACK扫描

这种扫描故意忽略TCP的三次握手顺序：
- 直接发送SYN/ACK数据包（而非先发SYN）
- 如果端口开放，目标主机返回RST信息
- 如果端口关闭，目标主机不返回任何信息

## 其他扫描技术

除了上述常见的端口扫描技术外，还有一些其他扫描技术：
- UDP扫描
- IP头信息dumb扫描
- 慢速扫描
- 乱序扫描

这些技术各有特点，适用于不同的网络环境和安全需求。

## 实验应用

在网络安全测试中，可以实现一款简单的端口扫描器，对特定的IP地址（范围）和特定的端口（范围）进行扫描，确定端口是否开放，并将结果显示在界面上。这种工具对于网络管理员和安全测试人员来说非常有用，可以帮助他们发现系统中潜在的安全漏洞。
