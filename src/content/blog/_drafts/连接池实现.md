# 连接池实现

目前一个基本的socket类已经封装好了。
之后要实现一个连接池。

连接池需要连接队列。

在线程队列的基础上完成一个连接池。

在连接数据库的过程中通常是先连接，最后的时候断开连接。

## Server

建立一个线程池，该池接收任务，并且提供触发器进行执行任务。

建立一个I/O模型完成端口。在船舶项目中利用 Windows 中的 WSA event 事件，将多个套接字socket分别与端口相对应，来达到监听是否有连接进入的问题。之后再分配独立次级主线程进行数据接收和发送。
而在本项目中只使用一个端口。建立一个线程进行监听，进行TCP连接。
如果连接成功就将该TCP连接关联到IOCP端口。
建立一个线程用于完成端口监听已经连接好的TCP连接的 send 请求。当有 send 信息时，根据关联信息，进行 WSARecv 收取数据。然后进行处理。
并行处理该数据，以一个方法的形式方便以后异步处理。解析数据包，然后将该包进行判断需要执行何种操作，回调该操作的指定函数，并分配给线程池进行运行执行。
在维持了一个TCP连接的基础上，不希望程序目前`与数据库的连接`经常性的主动断开。在连接成功数据库并执行完毕数据包所指向的操作之后，不进行释放，而是存放到有限的连接池中，维持一定时间，除非该操作为关闭数据库连接指令。

以上整个server的结构的第1版结构内容完成。
