# Skiplist Analyze

## Base Theory

在链表中，进行查找， 空间复杂度是 O(1)，时间复杂度是 O(n) ，


首先看一个问题：

* 为什么 Redis 中的有序集合用跳表而非红黑树来实现呢？[index link](https://blog.csdn.net/Appleeatingboy/article/details/119948340)
1. 对于插入，删除，查找 以及 输出有序序列 这几个操作，红黑树也可以完成，时间复杂度 与 用跳表实现是相同的。 但是，对于按照区间查找数据这个操作（比如 [20,300]）,红黑树的效率没有跳表高，跳表可以做到 O(logn)的时间复杂度定位区间的起点，然后在原始链表中顺序向后遍历输出，直到遇到值大于区间终点的节点为止。
2. 跳表更加灵活，它可以通过改变节点的抽取间隔，灵活地平衡空间复杂度和时间复杂度
3. 相比红黑树，跳表更容易实现，代码更简单。

而关键问题即是，在跳表中如何建立等级模型，是按照何种策略进行分布的。
本文将着重记录以下问题：
1. 策略方式
2. 时间复杂度和空间复杂度
3. 在数据库中的应用场景

### 策略方式

最基本的策略是利用 二分查找 的思想，每个等级都对上个等级进行间隔建立索引。但阅读 rust crate.io 库中的 skiplist ，其采用的是概率控制分布率的方式。也就是说，并不是严格的按照每两个元素中会有一个为最高级，而是采用随机数计算，使得其概率为 1/2 。

采用概率主要是因为，插入的时候需要在每一层进行判定，是否让插入的元素作为更高的等级索引，只要判断了一次 1/2 ， 那么接下来的概率都是 <1/4, 所以仍然可以控制分布在 1/2, 没有超过 log2(n) 的数量级。
