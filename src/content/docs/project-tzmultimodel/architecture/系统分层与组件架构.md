## 分层结构

1. UI 层：Qt 界面与交互页面。
2. 控制层：SQL 组装与业务动作转发。
3. 中间件层：统一 `middleware_*` 接口，屏蔽底层差异。
4. 驱动层：ODBC/JNI/JDBC 外部接入能力。
5. 服务部署层：服务端二进制与脚本化部署。

## 调用关系

```text
Qt GUI(view)
  -> Controller(SQL构建)
  -> Middleware API(middleware_*)
      -> parse_engine + mwkernel
      -> 底层数据库客户端能力
  -> ODBC/JNI/JDBC 工具链
```

## 架构结论

- 中间件层是核心边界，负责把 UI 需求映射为统一执行能力。
- 驱动链路独立成层，使该项目可同时服务 GUI 用户与外部程序接入。
