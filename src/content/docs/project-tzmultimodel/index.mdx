# TzMultiModel 项目功能与架构说明

## 1. 项目定位
`TzMultiModel` 是一个基于 Qt 的多模数据库管理客户端，目标是统一管理关系型与时序型数据能力，提供图形化操作界面、SQL 执行、数据导入导出、权限与账号管理、数据回放/校验、主题订阅、集群相关操作等能力。

从代码结构看，它不仅是一个 GUI 工具，还包含中间件、ODBC 驱动、JNI 桥接和 JDBC 驱动测试链路。

## 2. 核心功能
结合 `view/` 与 `model/middleware/include/Middleware.h` 的接口调用，当前主要功能包括：

- 数据库连接与会话管理（连接、断开、切库、元数据刷新）
- SQL 执行与结果展示（同步/异步查询、结果集游标、字段元信息）
- 元数据浏览（数据库/表结构导航）
- 表与类型相关操作（建库、建表、索引相关操作）
- 数据导入/导出（XML/XLS/CSV）
- 数据备份与恢复
- 用户账号与授权管理（用户增删改、授权）
- 数据同步与集群加入（`middleware_data_sync` / `middleware_join_cluster`）
- 数据回放与数据校验页面
- 主题订阅（生成订阅 SQL、消费订阅）
- 性能监控、索引管理等页面能力

## 3. 总体架构

### 3.1 分层结构
项目可以理解为 5 层：

1. 表现层（UI）: `view/`
2. 控制与 SQL 组装层: `controller/`
3. 中间件层（统一接口）: `model/middleware/`
4. 驱动与对外接口层: `tools/odbc`、`tools/odbc-jni`、`tools/TZMjdbc`
5. 服务端二进制与部署脚本: `server/`、`linux_install_server.sh`

### 3.2 调用关系（简化）

```text
Qt GUI(view)
  -> Controller(SQL 构建)
  -> Middleware C API(middleware_*)
      -> parser_engine(语句解析与引擎判定)
      -> mwkernel(环境/缓存/结果集/工具)
      -> dependence 中预编译库(db_client/kernel/sqlengine/...)
  -> ODBC Driver(tzmdb-odbc)
      -> JNI(odbc-jni)
      -> JDBC(TZMjdbc)
```

## 4. 模块组成

## 4.1 顶层模块

- `main.cpp`: Qt 应用入口，创建 `HomePage`。
- `view/`: 主界面与各业务页面（连接、SQL 编辑、回放、校验、导入导出、授权、订阅等）。
- `controller/`: 主要是 SQL 语句构建（`TzSQLQueryBuilder`）。
- `model/`: 包含 `middleware` 中间件子系统。
- `tools/`: ODBC/JNI/JDBC/安装器相关工具链。
- `server/`: Linux/ Debian 环境的服务端可执行文件与辅助文件。
- `test/`: ODBC 与 QtODBC 接口测试程序。
- `res/`: 图标、QSS、资源文件。
- `doc/`: 文档与截图资源。

## 4.2 View（界面层）

`view/CMakeLists.txt` 显示该层是体量最大的模块，包含：

- 主窗口与导航: `homepage.*`, `tzqtreeview.*`
- SQL 编辑与结果展示: `tzsqledit.*`, `sqleditwidget.*`, `tzresulttablewidget.*`
- 数据能力页面: `tzdatareplay.*`, `tzdataverify.*`, `tzperformance.*`
- 数据交换页面: `data_import.*`, `data_export.*`
- 账号与权限页面: `account_register.*`, `account_manager.*`, `grant_auth.*`
- 订阅能力页面: `sourcesubscribe.*`

该层通过 `middleware_*` 系列函数直接调用中间件。

## 4.3 Controller（控制层）

- `controller/sql/TzSQLQueryBuilder.cpp`
- `controller/include/sqlquerybuilder.h`

职责是封装建库、切库、建表、建类型等 SQL 字符串构建逻辑，降低界面层拼接 SQL 的复杂度。

## 4.4 Middleware（中间件层）

目录：`model/middleware/`

- `include/Middleware.h`: 对外 C 风格 API（连接、查询、结果集、元数据、导入导出、同步、授权、订阅等）
- `mwentry/`: 对外入口实现（如 `middleware.cpp`）
- `mwkernel/`: 环境、缓存、结果集、公共结构与工具实现
- `paser_engine/`（命名中存在 `parser` 拼写差异）:
- `lexer/` 词法
- `parser/` 语法
- `parser_test/` 解析测试

中间件内部通过 SQL 解析结果判定引擎类型（`ENGINE_TZDB/ENGINE_TSDB`），并路由到不同底层执行路径。

## 4.5 ODBC/JNI/JDBC 工具链

- `tools/odbc`: 构建 `tzmdb-odbc` 共享库（ODBC 驱动）
- `tools/odbc-setup`: ODBC 安装/配置相关静态库
- `tools/odbc-installer`: 安装器可执行程序
- `tools/odbc-jni`: JNI 动态库，桥接 Java 到 ODBC
- `tools/TZMjdbc`: Java JDBC 驱动实现（`TZMdbDriver` 等）

这部分使数据库可被 ODBC/JDBC 生态接入，不仅限于 Qt 客户端。

## 4.6 测试模块

- `test/interface-odbc`: 直接链接 `tzmdb-odbc` 的接口测试
- `test/interface-qtodbc`: 基于 QtSql 的 ODBC 测试
- `test_files/*.sql`: 功能与场景 SQL 脚本（统计、回滚、同步、导入导出等）

## 5. 技术栈与依赖

## 5.1 语言与构建

- C++20
- CMake（多子模块工程）
- Qt5（Core/Gui/Widgets/Charts/Sql）

## 5.2 关键依赖

- QScintilla（来自 `3rdparty/qscintilla`）
- ODBC / unixODBC（Linux 下链接 `odbcinst` 等）
- Java + Maven（`tools/TZMjdbc`）
- 预编译底层库（位于 `model/middleware/dependence`，如 `db_client/kernel/sqlengine`）

## 5.3 跨平台适配

CMake 中包含 `WIN32` 与 `UNIX/APPLE` 分支，分别处理：

- 编译宏
- 系统库（`ws2_32`, `odbc32`, `odbcinst` 等）
- 动态库导入与链接方式

## 6. 构建与运行概览

## 6.1 客户端构建

参考 `README.md`：

- 通过 CMake 配置 Qt5 路径
- `cmake -S . -B ./build ...`
- `cd build && make`

## 6.2 服务端部署（Linux）

`linux_install_server.sh` 会：

- 安装 `tzdb_server`/`tsdb_server` 到 `/usr/local/bin`
- 创建并启用 `tzdb_service.service`
- 通过 systemd 管理服务生命周期

## 6.3 ODBC 配置

`doc/odbc/odbc.md` 给出了驱动库部署路径和 `odbc.ini` / `odbcinst.ini` 示例。

## 7. 典型业务流程（从界面到执行）

1. 用户在 `HomePage` 连接数据库。
2. 界面层调用 `middleware_connect` 建立连接。
3. 用户在 SQL 编辑区执行语句，界面调用 `middleware_query`。
4. 中间件使用 `parse_engine` 解析 SQL，识别语句类型与引擎。
5. 中间件路由到底层关系/时序执行逻辑。
6. 返回 `MiddlewareResultSet`，界面层读取元数据与行数据并渲染结果表。

## 8. 当前工程特点与注意点

- 工程是“客户端 + 中间件 + 驱动链路 + 测试”一体化仓库，不是单一 GUI 项目。
- 中间件 API 设计较完整，覆盖了运维与数据治理能力（同步、备份、规则、订阅等）。
- 部分目录/命名存在历史痕迹（如 `paser_engine`），新开发时建议保持兼容并逐步规范。
- 多处依赖预编译库与本地环境（Qt、ODBC、服务端二进制），新环境搭建需优先确认依赖路径。
