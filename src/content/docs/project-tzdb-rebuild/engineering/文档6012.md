## 关键发现

1. **INSERT操作性能比较**:
   - 天脉分布式环境下的INSERT操作时间明显高于单节点环境，特别是在大数据包(>1KB)情况下
   - 有盘模式在天脉环境下比无盘模式性能更好，特别是在大数据包情况下
   - Windows环境在小数据包(<1KB)下性能优于天脉环境，但在大数据包下差距缩小

2. **QUERY操作性能比较**:
   - 查询操作在各环境下性能相对稳定，随数据包大小增长较为线性
   - 天脉无盘模式在某些数据包大小下(32B, 128B, 1024B等)出现异常高的平均查询时间
   - Windows环境下的查询操作最小时间普遍为0，可能与测量精度有关

3. **DELETE操作性能比较**:
   - 分布式环境下的DELETE操作时间明显高于单节点环境，且与数据包大小关系不大
   - 天脉分布式环境下的DELETE操作最大时间普遍在19ms左右，而单节点环境下仅为1ms左右
   - Windows环境下的DELETE操作性能明显优于天脉环境

4. **UPDATE操作性能比较** (仅单节点测试):
   - 天脉环境下的UPDATE操作性能随数据包大小增长较为线性
   - 天脉无盘模式和有盘模式在UPDATE操作上性能接近
   - Windows环境下的UPDATE操作数据不完整，但在有记录的情况下性能优于天脉环境

*注：
1. 用例5.3的64936B数据是从测试用例5.4中获取的，实际测试的是64512B
2. Windows环境下的最小时间为0可能是由于测量精度或记录方式导致的
3. 单节点测试(用例6.x)包含UPDATE操作，而分布式测试(用例5.x)没有此操作
4. 用例6.3和用例6.4在原始数据中都标记为"windows环境 无盘"和"windows环境 有盘"，但实际上是两个不同的测试用例

## 关键发现

1. **INSERT操作性能比较**:
   - 天脉分布式环境下的INSERT操作时间明显高于单节点环境，特别是在大数据包(>1KB)情况下
   - 有盘模式在天脉环境下比无盘模式性能更好，特别是在大数据包情况下
   - Windows环境在小数据包(<1KB)下性能优于天脉环境，但在大数据包下差距缩小

2. **QUERY操作性能比较**:
   - 查询操作在各环境下性能相对稳定，随数据包大小增长较为线性
   - 天脉无盘模式在某些数据包大小下(32B, 128B, 1024B等)出现异常高的平均查询时间
   - Windows环境下的查询操作最小时间普遍为0，可能与测量精度有关

3. **DELETE操作性能比较**:
   - 分布式环境下的DELETE操作时间明显高于单节点环境，且与数据包大小关系不大
   - 天脉分布式环境下的DELETE操作最大时间普遍在19ms左右，而单节点环境下仅为1ms左右
   - Windows环境下的DELETE操作性能明显优于天脉环境

4. **UPDATE操作性能比较** (仅单节点测试):
   - 天脉环境下的UPDATE操作性能随数据包大小增长较为线性
   - 天脉无盘模式和有盘模式在UPDATE操作上性能接近
   - Windows环境下的UPDATE操作数据不完整，但在有记录的情况下性能优于天脉环境

*注：
1. 用例5.3的64936B数据是从测试用例5.4中获取的，实际测试的是64512B
2. Windows环境下的最小时间为0可能是由于测量精度或记录方式导致的
3. 单节点测试(用例6.x)包含UPDATE操作，而分布式测试(用例5.x)没有此操作
4. 用例6.3和用例6.4在原始数据中都标记为"windows环境 无盘"和"windows环境 有盘"，但实际上是两个不同的测试用例
