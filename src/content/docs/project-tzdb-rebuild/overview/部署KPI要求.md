# 部署指标(KPI)与考核方式

> 说明：以下内容按“任务书及测试大纲 / 第三方测评或现场测试”口径整理。  
> 其中“7OKB”按原意统一为“70KB”。

| 序号 | 指标内容                                                                    | 考核依据     | 考核组织方式     | 考核方法                                                                        |
|----|-------------------------------------------------------------------------|----------|------------|-----------------------------------------------------------------------------|
| 1  | `(1) 管理内存开销`：最小功能集下，系统建立连接前后内存资源开销差值不超过 `70KB`。                         | 任务书及测试大纲 | 第三方测评或现场测试 | 在最小功能集下，记录建立连接前后内存占用，计算差值并判定是否 `<= 70KB`。                                   |
| 2  | `(2) 数据响应时间`：在 `100万` 条记录容量、单条记录 `<=16字节` 条件下，增/删/改/查单条平均响应时间不超过 `2us`。 | 任务书及测试大纲 | 第三方测评或现场测试 | 该指标拆分为 4 个独立测试(增/删/改/查)，每项各执行 `1万` 条记录并独立判定。                                |
| 3  | `(3) 查询吞吐量`：在 `100万` 条记录容量、单条记录 `<=16字节` 条件下，并发查询吞吐量不低于 `40万 QPS`。      | 任务书及测试大纲 | 第三方测评或现场测试 | 在并发查询场景下测试每秒可执行查询数量，判定是否 `>= 400,000 QPS`。                                  |
| 4  | `(4) 事务吞吐量`：在读写混合负载下，并发实时事务处理能力不低于 `10万 TPS`。                           | 任务书及测试大纲 | 第三方测评或现场测试 | 在同时包含读事务和写事务的并发场景下，测试每秒实时事务处理数，判定是否 `>= 100,000 TPS`。                       |
| 5  | `(5) 模型推理支持`：系统支持卷积、池化、全连接、激活函数等 `4` 种经典神经网络推理算子。                       | 任务书及测试大纲 | 第三方测评或现场测试 | 在不依赖 `PyTorch`、`TensorFlow` 等国外商用/开源推理框架条件下，验证系统可执行上述 `4` 类算子计算并支持简单智能模型推理。 |
| 6  | `(6) 平台兼容性`：适配主流国产机载硬件平台飞腾 `FT-2000` 系列处理器平台。                           | 任务书及测试大纲 | 第三方测评或现场测试 | 检查系统是否可在飞腾 `FT-2000` 系列处理器平台稳定运行。                                           |

## 指标2拆分(4个独立测试表)

### 2-1 数据增加(Insert)

| 项目   | 要求                        |
|------|---------------------------|
| 数据规模 | `100万` 条记录                |
| 记录约束 | 单条记录 `<=16字节`             |
| 测试操作 | 新增 `1万` 条记录               |
| 计时方式 | 记录开始时间与完成时间，`总时间 / 10000` |
| 判定标准 | 平均响应时间 `<= 2us`           |
