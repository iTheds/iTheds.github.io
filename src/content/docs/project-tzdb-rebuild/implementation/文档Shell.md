## 概述

TZDB Shell 是一个交互式命令行工具，用于连接和操作 TZDB 数据库。它提供了一个类似 SQLite CLI 或 PostgreSQL psql
的用户界面，允许用户通过命令行执行 SQL 查询、管理事务，以及与数据库进行交互。

## 核心功能

### 1. 数据库连接与初始化

- **灵活的数据库路径配置**：支持多种参数形式指定数据库位置
    - `shell` - 使用默认数据库 `./test_shell`
    - `shell mydb` - 使用 `./mydb`
    - `shell /data/db mydb` - 使用 `/data/db/mydb`
    - `shell /path/to/mydb` - 自动解析路径和数据库名

- **数据库初始化**：创建 `Database` 对象并初始化内部组件
    - 事务管理器(TransactionManager)
    - 数据目录(Catalog)
    - SQL 上下文(SqlContext)

### 2. 交互式查询执行

#### 查询输入处理

- **多行查询支持**：允许用户输入跨越多行的 SQL 语句
    - 以 `;` 或 `\` 开头的命令作为输入结束标志
    - 支持 TTY 和非 TTY 模式

- **查询规范化**：
    - 移除尾部空格和分号
    - 转换为小写用于命令识别
    - 清理空白字符

#### 查询执行模式

支持两种执行模式：

**自动事务模式**(无活跃事务)

```cpp
query::SqlContext sql_context(txn_manager, nullptr, catalog);
sql_context.PollExecuteGeneralSql(normalized_query, writer);
```

**显式事务模式**(有活跃事务)

```cpp
query::SqlContext sql_context(txn_manager, current_txn, catalog);
sql_context.Execute(normalized_query, writer);
```

### 3. 事务管理

Shell 支持完整的事务生命周期管理：

#### 事务命令

- `BEGIN` / `\BEGIN` - 开始新事务
- `COMMIT` / `\COMMIT` - 提交当前事务
- `ROLLBACK` / `\ROLLBACK` / `ABORT` / `\ABORT` - 回滚事务

#### 事务状态追踪

- **事务状态显示**：提示符中显示当前事务状态
    - `RUNNING` - 事务进行中
    - `COMMITTED` - 已提交
    - `ABORTED` - 已回滚
    - `TAINTED` - 被污染(执行出错)

- **事务 ID 显示**：人类可读的事务 ID(通过 XOR 运算转换)

#### 事务污染机制

- 当事务中的 SQL 执行失败时，事务被标记为 `TAINTED`
- 用户必须显式执行 `ROLLBACK` 才能清除污染状态
- 防止在错误状态下继续执行

### 4. 结果显示

#### FortTableWriter 类

使用 libfort 库将查询结果格式化为美观的表格：

- 支持表头显示
- 支持行列对齐
- 支持简化输出模式(无边框)
- 支持 UTF-8 编码

#### 输出特性

- 自动将 `Value` 对象转换为字符串
- 支持多个结果集的顺序输出
- 格式化的表格展示

### 5. 命令行选项

| 选项               | 说明                 |
|------------------|--------------------|
| `--emoji-prompt` | 使用浴缸 emoji 作为提示符   |
| `--disable-tty`  | 禁用 TTY 模式，用于非交互式输入 |
| `--no-txn`       | 禁用托管事务模式           |
| `--help, -h`     | 显示帮助信息             |

### 6. 交互式特性

#### 命令行编辑(linenoise-ng)

- 支持命令历史记录(最多 1024 条)
- 支持行编辑功能
- 支持窗口大小变化处理

#### 提示符

- 默认提示符：`tzdb> `
- 事务提示符：`txn <id>(<state))> `
- 续行提示符：`... `
- 可选 emoji 提示符：`🛁> `

## 工作流程

```
┌─────────────────────────────────────────────────────┐
│ 1. 启动 Shell                                        │
│    - 解析命令行参数                                  │
│    - 初始化数据库连接                                │
│    - 显示欢迎信息                                    │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ 2. 主循环                                            │
│    - 显示提示符(包含事务状态)                      │
│    - 读取用户输入(支持多行)                        │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ 3. 命令识别                                          │
│    ├─ 退出命令 (\q, quit, exit)                    │
│    ├─ 事务命令 (begin, commit, rollback)            │
│    └─ SQL 查询                                      │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ 4. 命令执行                                          │
│    ├─ 事务命令：调用 HandleTransactionalCmd()       │
│    └─ SQL 查询：调用 SqlContext.Execute()           │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│ 5. 结果显示                                          │
│    - 格式化为表格                                    │
│    - 输出到标准输出                                  │
│    - 错误输出到标准错误                              │
└─────────────────────────────────────────────────────┘
                        ↓
                    返回第 2 步
```

## 关键类和函数

### FortTableWriter

实现 `query::ResultWriter` 接口，用于将查询结果写入表格：

- `WriteCell()` - 写入数据单元格
- `WriteHeaderCell()` - 写入表头单元格
- `BeginHeader()` / `EndHeader()` - 表头边界
- `BeginRow()` / `EndRow()` - 行边界
- `BeginTable()` / `EndTable()` - 表格边界

### 主要函数

| 函数                                | 说明                            |
|-----------------------------------|-------------------------------|
| `NormalizeQuery()`                | 规范化查询字符串(移除空格和分号)             |
| `IsExitCommand()`                 | 检查是否为退出命令                     |
| `IsTransactionControlCommand()`   | 检查是否为事务控制命令                   |
| `HandleTransactionalCmd()`        | 处理事务命令(BEGIN/COMMIT/ROLLBACK) |
| `ReadQuery()`                     | 读取用户输入的查询(支持多行)               |
| `GetTransactionPrompt()`          | 生成包含事务状态的提示符                  |
| `GetTransactionIdHumanReadable()` | 转换事务 ID 为人类可读格式               |

## 使用场景

### 1. 数据库开发和测试

- 快速执行 SQL 查询
- 验证数据库功能
- 调试 SQL 语句

### 2. 交互式数据探索

- 浏览数据库表结构
- 查询数据内容
- 执行临时操作

### 3. 事务测试

- 测试事务隔离性
- 验证 ACID 属性
- 调试并发问题

### 4. 数据库管理

- 执行 DDL 操作(创建表、索引等)
- 执行 DML 操作(插入、更新、删除)
- 管理数据库对象

## 错误处理

### 异常捕获

- 捕获 SQL 执行异常
- 显示错误信息到标准错误
- 将事务标记为 TAINTED

### 事务污染

- 执行失败时自动标记事务为 TAINTED
- 防止在错误状态下继续执行
- 强制用户显式回滚

### 清理机制

- 退出时检查活跃事务
- 自动回滚未提交的事务
- 释放历史记录资源

## 特殊设计

### 1. 事务 ID 转换

```cpp
const tzdb::txn_id_t TXN_START_ID = 1LL << 62;
auto GetTransactionIdHumanReadable(tzdb::txn_id_t txn_id) 
    -> tzdb::txn_id_t { 
    return txn_id ^ TXN_START_ID; 
}
```

通过 XOR 运算将内部事务 ID 转换为更易读的形式。

### 2. 双模式执行

- **自动事务模式**：每个查询自动包装在事务中
- **显式事务模式**：用户显式管理事务生命周期

### 3. 多行输入支持

- 检测行尾是否为 `;` 或 `\`
- 支持跨越多行的长查询
- TTY 和非 TTY 两种输入方式

## 总结

TZDB Shell 是一个功能完整的数据库交互工具，具有以下特点：

✅ **易用性** - 提供友好的命令行界面和交互式体验  
✅ **功能完整** - 支持 SQL 执行、事务管理、结果格式化  
✅ **灵活配置** - 支持多种命令行选项和参数组合  
✅ **健壮性** - 完善的错误处理和事务污染机制  
✅ **可扩展性** - 基于接口设计，易于扩展功能

它是开发者与 TZDB 数据库交互的主要工具，类似于 SQLite 的 CLI 或 PostgreSQL 的 psql。
