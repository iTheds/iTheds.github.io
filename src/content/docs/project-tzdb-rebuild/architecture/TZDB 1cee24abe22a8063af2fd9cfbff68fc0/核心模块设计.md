核心模块负责数据库的基本操作和管理策略，包括表的创建、删除、查询等功能。在重构版本的`kernel`
实现中，它是战略上的而非战术上的，是一个策略集而不是实现集，旨在能够从数据库最基本的功能出发来完全抽象数据库内核。

所以，按照以上的思路，以存储为例，对于内核的实现大概应该为这样:

```cpp
class DB {
public:
	bool init();
	bool open();
	
	int create_table(std::string table_name, Attrs attrs);
	/* ... */
private:
	struct Impl;
	Impl* impl_;
};

struct DB::Impl {
    std::vector<StorageInterface*> storages_;
		TransactionInterface* trxs_;
};

/* 省略无关代码 */

// 创建表.
int DB::create_table(std::string table_name, Attrs attrs) {

}
```

这种写法使得内核更像是一个策略的集合。
