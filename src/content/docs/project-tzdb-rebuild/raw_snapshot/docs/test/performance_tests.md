# 数据库性能测试用例

## 1. 内存开销需求测试 (最小功能集 - tzdb_minimal 变体)

参考：tzdb_minimal_test.cpp

| 功能追溯 | DSS-XN-1 |
|----------|----------|
| 用例名称 | 内存开销需求测试 |
| 用例标识 | DSS_MEMORY_OVERHEAD |
| 测试项标识 | DSS_PERFORMANCE |
| 初始化 | 目标机和宿主机已就绪，tzdb_minimal变体已编译完成，支持内存监控功能。 |
| 前提 | 系统支持最小功能集运行(基础API操作，无SQL解析器和查询优化器)。 |
| 终止条件 | 被测试项出现故障或测试取得预期结果。 |
| 用例期望 | tzdb_minimal变体下系统运行时总内存开销不超过70KB。 |
| 评估准则 | 系统运行时内存占用 ≤ 70KB。 |

| 步骤 | 描述 | 预期结果 | 评估准则 |
|------|------|----------|----------|
| 1 | 编译tzdb_minimal变体：使用CMake构建tzdb_minimal库，仅包含基础API功能 | 编译成功，无错误，生成tzdb_minimal库文件 | 与预期结果一致 |
| 2 | 分配最小内存块：调用malloc分配70KB内存用于数据库实例 | 内存分配成功，指针非空 | 与预期结果一致 |
| 3 | 测量初始内存占用：使用系统工具测量进程启动后的内存使用量 | 内存测量成功，记录初始值m0 | 与预期结果一致 |
| 4 | 初始化数据库实例：调用edb_db_open_base创建数据库实例，使用分配的内存块 | 数据库初始化成功，返回EDB_OK | 与预期结果一致 |
| 5 | 建立数据库连接：调用edb_db_connect建立连接句柄 | 连接建立成功，返回EDB_OK | 与预期结果一致 |
| 6 | 启动事务：调用edb_trans_start开始读写事务 | 事务启动成功，返回EDB_OK | 与预期结果一致 |
| 7 | 执行基础API操作：创建记录对象，设置整数和字符串字段，提交事务 | API操作成功，所有调用返回EDB_OK | 与预期结果一致 |
| 8 | 断开连接并关闭数据库：调用edb_db_disconnect和edb_db_close | 断开和关闭成功，无错误 | 与预期结果一致 |
| 9 | 测量运行后内存占用：再次测量进程内存使用量 | 内存测量成功，记录结束值m1 | 与预期结果一致 |
| 10 | 计算内存开销：计算m1 - m0得到运行时内存开销 | 计算成功，开销值 ≤ 70KB | 与预期结果一致 |
| 11 | 验证内存限制：检查内存开销是否不超过70KB | 验证通过，内存开销在限制范围内 | 与预期结果一致 |
| 12 | 释放内存：调用free释放分配的内存块 | 内存释放成功，无泄漏 | 与预期结果一致 |

## 2. 响应时间需求测试

参考：mem_sql_performance_test.cpp

| 功能追溯 | DSS-XN-2 |
|----------|----------|
| 用例名称 | 响应时间需求测试 |
| 用例标识 | DSS_RESPONSE_TIME |
| 测试项标识 | DSS_PERFORMANCE |
| 初始化 | 目标机和宿主机已就绪，数据库已创建，支持高性能测试环境。 |
| 前提 | 系统支持百万级数据存储，每条记录不超过16字节。 |
| 终止条件 | 被测试项出现故障或测试取得预期结果。 |
| 用例期望 | 百万记录数据容量下，单条数据增删改查平均响应时间不超过2us。 |
| 评估准则 | 平均响应时间 ≤ 2us。 |

| 步骤 | 描述 | 预期结果 | 评估准则 |
|------|------|----------|----------|
| 1 | 创建测试表：执行SQL CREATE TABLE perf_test (id INT PRIMARY KEY, data VARCHAR(16)) | 表创建成功 | 与预期结果一致 |
| 2 | 批量插入百万条数据：循环插入1000000条记录，每条数据≤16字节 | 插入成功，表包含1000000条记录 | 与预期结果一致 |
| 3 | 预热系统：执行几次查询和更新操作，确保系统稳定 | 预热完成，无异常 | 与预期结果一致 |
| 4 | 测量插入响应时间：执行1000次随机INSERT操作，记录总时间t_insert | 测量成功，t_insert > 0 | 与预期结果一致 |
| 5 | 测量删除响应时间：执行1000次随机DELETE操作，记录总时间t_delete | 测量成功，t_delete > 0 | 与预期结果一致 |
| 6 | 测量更新响应时间：执行1000次随机UPDATE操作，记录总时间t_update | 测量成功，t_update > 0 | 与预期结果一致 |
| 7 | 测量查询响应时间：执行1000次随机SELECT操作，记录总时间t_select | 测量成功，t_select > 0 | 与预期结果一致 |
| 8 | 计算平均响应时间：计算t_insert/1000, t_delete/1000, t_update/1000, t_select/1000 | 计算成功，各平均值均 ≤ 2us | 与预期结果一致 |
| 9 | 验证时间限制：检查所有平均响应时间是否不超过2us | 验证通过，所有操作满足时间要求 | 与预期结果一致 |
| 10 | 清理测试数据：删除测试表DROP TABLE perf_test | 表删除成功 | 与预期结果一致 |

| 2 | 配置并发测试环境：设置并发连接数为100，查询语句为SELECT * FROM throughput_test WHERE id = ? | 环境配置成功 | 与预期结果一致 |
| 3 | 预热系统：执行少量并发查询确保系统稳定 | 预热完成，无异常 | 与预期结果一致 |

| 6 | 验证吞吐量要求：检查QPS是否不低于40万 | 验证通过，吞吐量满足要求 | 与预期结果一致 |
| 7 | 收集性能指标：记录CPU使用率、内存使用率、响应时间分布 | 指标收集成功 | 与预期结果一致 |
| 8 | 清理测试环境：删除测试表和数据 | 清理完成，无残留数据 | 与预期结果一致 |
