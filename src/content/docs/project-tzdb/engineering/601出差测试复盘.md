## 任务

主要解决上一阶段，插入数据缓慢问题，上次问题报告如下:
<待补充>

本次收到底层设备通信研发人员建议，采用 udp 协议进行通信；并且数据库进行通信重构，本次出差旨在测试天脉操作系统下数据库 udp 网络表现性能，其他测试均为参照测试。

主要测试以下内容:
1.单条消息增删改查了落盘时间，写入内存时间，平均值和极值(16字节-64K)
2.单条消息网络同步时间，测一下10000包数据的平均值及极值(16字节-64K)

其数据量要求为 16B - 64 KB，对极值与 2 倍阶梯值进行测试；

以下均为不同 CPU 测试:
1. 测试 udp socket 一发一收，发送时间；
2. 测试 udp socket 回传测试，记录为回传时间(单次通路时间 = 回传时间/2)；
3. 测试 udp CONNECT api 回传测试表现；
4. 测试 udp net_pool api 回传表现；
5. 测试 udp 数据同步下 CURD 时间；
6. 测试 单节点下，有盘模式下，数据的 CURD 时间；
