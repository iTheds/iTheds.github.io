## 调研范围

原始记录虽然简短，但主题并不空泛，主要围绕以下几个方向展开:

- 内存池 `buffer pool`
- 文件系统设计
- 页设计
- 内存索引设计
- 线程池与 checkpoint

原稿同时给出了一些取舍判断:

- 异步 I/O 可先不做。
- `redo log` / `undo log` 可暂不作为第一阶段重点。
- 线程池更适合承担定期刷盘、释放 `undo log`、实现 checkpoint 的职责。

## 关注的对象

原稿把存储引擎拆成几个最基本的对象:

- 数据页
- 索引页
- 现有影子页

对应的关键问题是:

- 如何通过地址直接定位数据页。
- 如何通过索引定位目标页并执行修改。
- 一条 `update` 语句怎样被完整执行。

## 当前结论

这一份记录还处于调研起步阶段，但已经把问题边界画出来了:如果后续真要做存储引擎优化，不能只看页结构，也必须同时考虑索引、刷盘、checkpoint 和内存管理的整体路径。
