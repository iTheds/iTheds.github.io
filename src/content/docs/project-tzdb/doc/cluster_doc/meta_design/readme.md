## 

元数据（MetaData），是指定义数据结构的数据。那么数据库元数据就是指定义数据库各类对象结构的数据。 例如数据库中的数据库名，表明， 列名、用户名、版本名以及从SQL语句得到的结果中的大部分字符串是元数据

问题变成了是否要落盘，是否要同步，同步类型是什么有哪些需要同步。

1. 同步：
   1. 网络元信息是否需要同步，在 ddl 配置读取情况下， 取决于网络是否发生变更；
      1. 网络变化的情况是
         1. 有新增节点或者删除节点；
   2. 库表元信息是否需要同步，在 ddl 配置读取情况下， 取决于主从节点所存储的库表是否发生变更；
      1. 同步是否需要事务；
         1. 是否存在冲突；冲突不能解决情况；
            1. 是否需要将第三个节点的元数据信息发送到(询问/同步信息)节点；
2. 落盘：
   1. 网络元信息是否需要落盘，在 ddl 配置读取情况下， 取决于；
      1. 宕机恢复情况
      2. 重启后如何构建网络：
         1. 即构建网络后，不完全依赖 ddl 配置读取
   2. 库表元信息是否需要落盘，在 ddl 配置读取情况下， 取决于主从节点所存储的库表是否发生变更；
      1. 节点是否要存储其他节点的库表元信息；
         1. 如何获取其他节点的库表元信息：
            1. ddl 记录；
            2. 同步；
         2. 重启后如何构建元数据信息；
            1. 本节点的元信息；
               1. ddl 记录
            2. 其他节点的元信息；
               1. ddl 记录
               2. 同步；
