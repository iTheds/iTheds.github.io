﻿# 主节点设计
## 简介
主节点提供处理事务请求、同步事务日志的功能。在整个集群中，主节点负责控制整个集群的事务，主节点的事务也同样是整个集群的事务，集群中各节点的事务策略依靠主节点的数据库内核中的事务调度策略来完成。  
## 类图
和`从节点`相同，主节点继承了`CLNode`类，重写了`CLNode`类中`处理开启事务请求`、`处理提交事务请求`的方法;与`CLNode`不同的是，添加了一个缓冲区用来管理`节点id->事务指针`这样的一个`pair`，目的是让`事务指针`和`节点`之间有一个可供查询的对应表，当事务执行完成时，可以将结果发送至指定节点。  
## 外部调用
主节点以来一下模块来完成元数据查询、网络传输、事务控制的功能：  
- kernel (事务)
- meta (元数据)
- transport (网络传输)
