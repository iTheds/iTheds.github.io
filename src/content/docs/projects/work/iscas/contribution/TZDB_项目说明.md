# TZDB 项目说明 by iTheds

## 项目概述

参与 TZDB 数据库服务端与工具链研发，负责远程访问服务、实时数据分发模块及 ODBC 驱动体系建设。工作覆盖网络通信、协议设计、SQL 执行链路、诊断与安装交付，支撑数据库在服务化访问与外部生态接入上的工程落地。

## 负责模块

- `raserver`：数据库远程访问服务核心模块
- `tools/RecGH`：实时数据接入/分发/解析/存储模块
- `tools/tzdb-odbc`、`tools/tzdb-odbc-setup`、`tools/tzdb-odbc-installer`：ODBC 驱动、配置组件与安装程序

## 核心工作内容

### 1. raserver（远程访问服务）

- 参与服务端请求处理链路开发：连接接入、指令解析、SQL 执行、结果封装与回包。
- 参与协议对象化设计（`DataInfoFormat` 系列），支持执行类、元数据类、监控类、导入导出类请求的统一编解码。
- 参与网络与 I/O 抽象层实现（`NetPool` / `IOMode` / `ProtocolFormat`），提升连接管理与协议处理可复用性。
- 参与客户端接口层（`libtzdb`）调用链建设，形成 connect/open/execute/fetch/meta 等标准化访问能力。

编程设计与功能效益：采用“网络层-协议层-执行层-接口层”分层设计，将远程访问能力标准化为可复用服务组件；通过统一请求处理链和协议对象化封装，提升了服务端扩展性与问题定位效率，降低了上层应用接入复杂度。

### 2. RecGH（实时分发模块）

- 参与发布-订阅-消费模型实现，构建 `Publisher/Subscriber/Consumer` 的异步处理流水线。
- 参与多协议数据通道建设（TCP/UDP/组播），实现多源数据接入与按目标地址分发。
- 参与消息结构（`FaiLInfo`）设计与处理流程，实现“接收-分发-解析-存储”一体化链路。
- 参与数据解析与落库接口对接（含 AIS/NMEA 相关处理链）。

编程设计与功能效益：采用发布-订阅-消费解耦架构，把接入、路由、处理、存储拆分为可独立演进的模块；在多协议实时链路下提升了系统可扩展性和稳定性，能够更低成本接入新数据源与新消费链路。

### 3. ODBC 驱动体系（tzdb-odbc*）

- 参与 `tzdb-odbc` 驱动核心实现，覆盖环境/连接/语句/描述符句柄管理与主要 ODBC API 适配。
- 参与分布式连接扩展与连接字符串解析能力建设（含 `DISTRIBUTED=TRUE` 相关链路）。
- 参与诊断体系实现（`SQLGetDiagRec/SQLGetDiagField` 及 W 版本扩展），完善错误可观测性与兼容性。
- 参与 `tzdb-odbc-setup` 与 `tzdb-odbc-installer` 组件开发，完善驱动配置、安装与交付流程。

编程设计与功能效益：基于 ODBC 标准接口实现数据库适配层，并补齐分布式连接、诊断接口和安装交付链路；提升了驱动兼容性与可运维性，缩短了第三方工具和业务系统接入 TZDB 的路径。

## 个人工作总结（可用于简历收尾）

在该项目中，我的工作重点是把数据库内核能力通过稳定的服务端与驱动接口对外输出，打通“网络接入 -> 协议处理 -> SQL 执行 -> 结果返回 -> 外部驱动接入”的完整工程链路；同时在实时分发模块中承担数据通道与处理模型实现，提升了系统在多协议、多模块协同下的可扩展性与可维护性。
