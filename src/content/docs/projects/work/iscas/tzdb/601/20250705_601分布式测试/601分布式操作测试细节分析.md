## 关键发现

1. **总时间对比**:
   - 天脉环境的总操作时间显著高于Windows环境，特别是在INSERT和DELETE操作上
   - 天脉环境下的INSERT操作随数据包大小增长明显，而Windows环境增长较为平缓
   - 天脉环境下的DELETE操作时间与数据包大小关系不大，基本保持在8000-11000μs范围内

2. **开始时间对比**:
   - 天脉环境的操作开始时间普遍高于Windows环境，表明操作准备阶段消耗更多时间
   - 天脉环境的INSERT操作开始时间在1000-6600μs范围内，而Windows环境仅在70-700μs范围内
   - QUERY操作的开始时间在两个环境中差异较小

3. **执行时间对比**:
   - INSERT操作的执行时间随数据包大小增长，两个环境表现相似，但天脉环境略高
   - 天脉环境的QUERY执行时间随数据包大小线性增长，而Windows环境波动较大且普遍较低
   - DELETE操作在天脉环境下执行时间随数据包大小增长，而Windows环境下基本保持较低水平

4. **提交时间对比**:
   - 天脉环境的INSERT和DELETE操作提交时间显著高于Windows环境
   - 天脉环境的INSERT提交时间在3700-20400μs范围内，而Windows环境仅在320-5210μs范围内
   - 天脉环境的DELETE提交时间在3100-6350μs范围内，而Windows环境仅在200-465μs范围内

5. **异常数据**:
   - 天脉环境64KB数据的INSERT操作执行时间异常低(1074μs)，可能是测试数据记录错误
   - Windows环境下的QUERY提交时间全部为0，可能与测量方法或记录方式有关

*注：
1. 天脉环境的64KB左右数据实际是63488字节，而Windows环境是65476字节
2. 天脉环境没有UPDATE操作的测试数据
3. Windows环境下的很多时间值为0，可能是由于测量精度或记录方式导致的
