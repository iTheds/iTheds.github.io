## 时间范围

- 2024-07 至 2024-12

## 实现主线

### 2024-07：通信层与协议分工

- 在 `CONNECT` 层统一协议处理职责，区分 TCP 粘包处理与 UDP 分包重传处理。
- 明确 `netSocket` 作为基础适配层，屏蔽 Linux/Windows 操作差异。

### 2024-08 至 2024-09：分包与重传机制

- 设计大包拆分与包标记机制，支持按片段回执缺失包并定向补发。
- 建立“回执等待窗口 + 失败判定 + 重发节流”流程，以降低丢包影响。
- 将链路质量反馈与发送节奏联动，动态调整发送频率和时间窗口。

### 2024-10 至 2024-11：任务队列与可观测性

- 将任务分发抽象为多级多类任务队列，按带宽和优先级选择压缩策略。
- 记录发送长度、压缩后长度、算法类型和失败次数，支撑状态监控展示。
- 在图谱与连接状态展示中加入上线/下线与连接质量可视化。

### 2024-12：外场联调与问题收敛

- 完成与金美接口联调，支持图谱获取、离线检测与数据回放。
- 修复连接关闭后 `io model` 残留操作符导致的循环处理问题。
- 完成多跳通信临时方案，基于路由信息在节点间继续转发任务。

## 关键实现结论

- 自适应分发需要与链路质量测量强绑定，仅靠静态策略难以稳定满足指标。
- 重传机制必须最小化全量重发，缺失片段回补是低带宽场景下的关键收益点。
- 外场演示中“接口可用”不等于“展示有效”，可观测性和界面表达直接影响验收观感。
