## 时间锚点

- 对接与重构阶段：2025-03 至 2025-04

## 数据建模要点

- 前端数据集与后端数据集分离维护，字段映射在请求层集中管理。
- 采集统计结构围绕 `totalValue`、`totalCount` 与单位定义展开。
- 字典数据、用户数据、日志数据分别按持久化策略存放。

## 请求层演进

- `backEndRequest`：底层请求封装。
- `makeRequestWithErrorHandling`：统一错误处理与结构标准化。
- 业务函数层：返回可直接用于页面渲染的业务结构。

## 问题与处理

- 典型问题：超时、内部错误、字段不一致。
- 处理方式：将网络错误与业务错误分层处理，降低页面层复杂度。
