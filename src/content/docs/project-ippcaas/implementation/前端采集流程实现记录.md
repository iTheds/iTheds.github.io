## 时间锚点

- 主要实现阶段：2025-02 至 2025-04

## 实现主线

- 创建日志：确定采样方法、采样区域数量与基础信息。
- 区域采集：围绕 p_log 建模，支持多区域并行采集。
- 病虫记录：按病害/虫害/天敌记录分层录入，并支持拍照附件。
- 提交流程：支持创建、暂存、提交与按 key 同步。

## 关键组件

- `CollectionInterface`：采集主流程容器。
- `CropCollection`：作物维度采集。
- `PestDiseaseCollection`：病虫害维度采集。

## 实现经验

- 前端数据模型需与界面渲染模型分离，避免耦合导致的维护成本。
- 多角色协同采集场景下，日志与区域两层结构优于单一列表结构。
