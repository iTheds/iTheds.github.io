# 文档projectippcaas

## 1. 项目概述

IPP-CAAS(农业监测预警平台)是一个面向植物保护与农业监测场景的数字化系统，聚焦“数据采集 - 业务分析 - 预警服务”闭环。
项目采用前后端分离架构:

- 移动端/多端前端:`uni-app + Vue3`(支持 H5、小程序、App 等)
- 服务端:`Golang + Gin + GORM`
- 数据与中间件:`MySQL + Redis`

当前代码基线中，系统标题为“盐碱地重大病虫害监测预警数字化平台”，重点覆盖病虫害监测日志、基础字典、气象与测报数据接入等能力。

## 2. 建设目标与业务价值

### 2.1 建设目标

- 将传统线下监测记录流程标准化、电子化
- 打通区域、监测点、作物、病虫害等基础数据
- 形成可追溯的监测日志与统计分析基础
- 为预警发布、订阅、地图展示等业务提供数据支撑

### 2.2 业务价值

- 减少人工填报与汇总成本，提高监测效率
- 提升数据一致性和可检索性
- 支持跨区域、跨作物维度的快速筛查与分析
- 为监管和科研提供结构化沉淀数据

## 3. 功能模块总览

### 3.1 前端业务模块(应用中心)

按页面配置与菜单分组，系统提供以下业务入口:

- 监测业务:监测日志、业务查询、气象日志、统计分析
- 预警业务:协同预警、预警发布、预警订阅、预警地图
- 咨询业务:离线咨询、离线诊断、协同诊断、智能识别
- 基础信息:区域管理、监测点、监测设备
- 知识管理:分类鉴定、防控知识、预警知识
- 通用模块:登录注册、待办事项、个人中心、设置、意见反馈、帮助中心

### 3.2 后端核心接口能力

后端接口分组围绕 `tools / user / monitor-data / common-data / sensor-data`:

- 认证与账号:登录(含验证码)、无验证码登录、注册、用户信息维护
- 监测日志主流程:
- 总日志(mlog)新增/更新/提交/删除
- 区域日志(plog)新增/更新/查询/删除
- 病虫害/病害/天敌日志(dplog/pestlog/diseaselog/nelog)增删改查
- 各类日志的附件上传与删除
- 公共基础数据:
- 区域、监测点、作物、病虫害/天敌名称与详情
- 多作物字典(棉花、玉米、大豆、水稻、田菁、苜蓿等)
- 本地文件访问接口
- 传感器数据接入:
- 气象数据写入
- 测报数据写入
- 测报图片数据写入

## 4. 技术架构说明

### 4.1 架构分层

```text
uni-app 前端(多端)
   ↓ HTTP/JSON + x-token
Gin Router(路由/中间件)
   ↓
API 层(参数绑定、响应封装)
   ↓
Service 层(业务编排)
   ↓
Model 层(GORM 模型)
   ↓
MySQL(业务数据) + Redis(限流/会话/缓存)
```

### 4.2 服务端技术要点

- Web 框架:Gin
- ORM:GORM(MySQL)
- 配置管理:Viper(`config.yaml`)
- 鉴权:JWT(`x-token`)，支持黑名单与续签
- 中间件:CORS、限流(基于 Redis 计数)
- 文档:Swagger(`/swagger/index.html`)
- 实时通信能力:已包含 WebSocket 服务实现(可按路由策略启用)

### 4.3 前端技术要点

- 框架:Vue3 + uni-app(Vite 构建)
- 状态管理:Vuex
- 多端脚本:H5、微信小程序、App 等统一命令
- 请求封装:统一 `request` 层，支持 token 注入、错误处理、上传封装
- UI 组织:按页面分包，业务模块化拆分(日志、图表、表单、地图组件等)

## 6. 部署与运行说明

### 6.1 后端(server)

依赖:MySQL、Redis、Go(建议使用 `go.mod` 对应版本链)。
基础步骤:

1. 配置 `server/config.yaml`(数据库、Redis、服务端口等)
2. 初始化依赖并启动:

```bash
cd server
go mod tidy
go run main.go
```

默认端口配置为 `8888`(可在配置文件调整)。

### 6.2 前端(uni3client)

基础步骤:

```bash
cd uni3client
npm install
npm run dev:h5
```

后端地址在 `src/config/index.config.js` 中配置，演示前需替换为目标环境地址。

## 7. 展示建议(对外汇报可直接使用)

### 7.1 建议演示路径(5-10 分钟)

1. 登录进入系统，展示“应用中心”业务全景
2. 新建监测日志(总日志 -> 区域日志 -> 病虫害记录)
3. 上传图片并提交日志
4. 在“监测日志列表/业务查询”中按作物、地区、时间筛选
5. 展示字典与基础数据接口(区域、作物、病虫害)
6. 说明预警与咨询模块可扩展能力

### 7.2 推荐展示亮点

- 多端统一开发，一套代码可覆盖多终端
- 监测日志模型完整，支持分层数据与附件管理
- 后端接口清晰、Swagger 可视化，便于联调与对外对接
- 支持 JWT、限流等基础安全与治理能力

## 8. 当前版本边界与后续规划

### 8.1 当前版本边界

- 部分业务页面已完成入口与交互框架，深度业务逻辑仍可继续完善
- WebSocket 订阅能力在代码中已具备实现，可按业务需要开放路由
- 配置文件中存在示例连接信息，生产部署需统一替换与加密管理

### 8.2 后续规划建议

- 增强预警规则引擎与自动触发机制
- 建设可视化看板(时序趋势、空间热力、预警分级)
- 引入更细粒度的角色权限与审计日志
- 完善自动化测试与持续集成流程
