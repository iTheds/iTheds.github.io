## 时间锚点

- 架构整理时间范围：2025-01 至 2026-03

## 架构分层

```text
uni-app 前端(多端)
   ↓ HTTP/JSON + x-token
Gin Router(路由/中间件)
   ↓
API 层(参数绑定、响应封装)
   ↓
Service 层(业务编排)
   ↓
Model 层(GORM 模型)
   ↓
MySQL + Redis
```

## 技术栈

- 前端：`uni-app + Vue3`，覆盖 H5/小程序/App 多端。
- 后端：`Golang + Gin + GORM`。
- 数据与中间件：`MySQL + Redis`。
- 鉴权与治理：`JWT(x-token)`、跨域中间件、限流中间件。

## 架构结论

- 该项目采用“多端统一前端 + 轻量后端服务 + 关系型数据存储”的组合。
- 业务边界清晰，适合以日志采集、预警发布、字典查询为核心的农业监测场景。
