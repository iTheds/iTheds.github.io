# 弱网条件模拟

通过建立本地回环，并且对本地回环进行限制，以达到模拟弱网的效果。

## 建立本地回环

```shell
ip addr show lo
sudo ip addr add 127.0.0.2/8 dev lo
sudo ip addr add 127.0.0.3/8 dev lo
sudo ip addr add 127.0.0.4/8 dev lo
```

## 添加限制规则

在 Linux 中，可以通过一些工具和方法对本地回环接口（lo）的带宽进行限制。这通常用于测试和模拟网络条件。以下是几种常用的方法：
1. 使用 tc (Traffic Control)

tc 是 Linux 中一个强大的流量控制工具，可以用来设置带宽限制、丢包、延迟等。
设置带宽限制示例

下面是一个示例，展示如何使用 tc 来限制 lo 接口的带宽：

    清除已有的规则：

    `sudo tc qdisc del dev lo root`

    添加一个根队列：

    `sudo tc qdisc add dev lo root handle 1: htb default 11`

    创建一个类来限制带宽：

    这里设置带宽为 1Mbps：

    `sudo tc class add dev lo parent 1: classid 1:1 htb rate 1mbit`

    添加过滤器（如果需要）：

    `sudo tc filter add dev lo protocol ip parent 1:0 prio 1 u32 match ip dst 127.0.0.1 flowid 1:1`

2. 使用 iptables 和 tc

结合 iptables 和 tc，可以对特定流量进行限制。例如，可以使用 iptables 标记流量，然后用 tc 进行带宽限制。
3. 使用 netem

netem 是 tc 的一部分，专门用于网络仿真，可以设置延迟、丢包等。你可以在带宽限制的同时，设置延迟和丢包。
清除已有规则

sudo tc qdisc del dev lo root
添加 netem 规则

sudo tc qdisc add dev lo root netem rate 1mbit
验证效果

设置完成后，你可以使用 iperf 或 netcat 等工具测试带宽效果。例如，你可以在一端启动一个 nc 服务器：

nc -l -p 12345 > /dev/null

然后在另一端进行测试：

dd if=/dev/zero bs=1M count=10 | nc 127.0.0.1 12345

这样你可以看到带宽限制的效果。
注意事项

    使用 tc 进行带宽限制时，确保对流量有足够的了解，以免误操作影响到其他应用程序。
    记得在测试完成后，使用 tc qdisc del dev lo root 来清除设置。
