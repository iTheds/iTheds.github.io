# 项目概述

## 项目定位

本项目是一个基于 Rust 的时序数据库工程，目标是建设具备单机原型与分布式扩展能力的 TSDB 系统。项目不仅关注时序数据写入、查询与管理，也覆盖客户端接入、服务端执行、流式处理、订阅消费和集群能力。

## 时间锚点

- 项目开发记录起点：2022-10
- Rust 方向正式开发：2023-02
- 订阅消费能力持续推进至：2024-10

## 目标范围

- 支持时序数据写入、查询与基础管理。
- 支持多节点响应与分布式扩展。
- 提供 CLI 与远程访问能力。
- 具备 Topic、Consumer、Subscribe 等订阅消费语义。
- 配套逻辑测试、服务测试、磁盘性能测试与基准测试能力。

## 核心组成

- 基础系统层：共享内存、休眠、线程、文件、socket 等系统抽象。
- 工具与算法层：跳表、红黑树、错误码、缓存与通用容器。
- 公共能力层：SQL 解析、逻辑计划、结果集、协议与消息结构。
- 核心引擎层：查询执行、服务引擎、时序存储、WAL、元数据。
- 接入层：CLI、客户端 SDK、RPC 与后续 Java/ODBC 方向接口。

## 关键技术方向

### 查询与执行

- 通过 SQL 解析、逻辑计划与物理执行链路实现查询能力。
- 在服务端围绕消息体系组织执行过程，形成统一的请求与响应流转。

### 流式与订阅消费

- 引入 Topic、Consumer、Subscribe 等对象，支持增量数据获取。
- 订阅流程由元数据侧保存定义，再向执行侧部署具体消费结构。

### 集群与元数据

- 集群模块承担节点、库表、分区与部署信息的组织与检索。
- 元数据用于数据路由、节点定位和部署决策，是分布式执行的关键支撑。

### 存储与可靠性

- 关注 WAL、磁盘引擎、时序数据落盘与恢复机制。
- 在后期阶段持续推进同步、复制与 IO 模型适配。

## 工程特征

- 该项目不是单一数据库内核，而是“基础设施 + 引擎 + 服务 + 测试”的一体化工程。
- 原稿中保留了大量设计推演、开发日志与问题排查记录，反映出项目是沿着“先基础设施、再执行框架、再订阅消费与集群”的路径逐步推进的。

## 总结

TSDB Rust 项目是一条从基础组件、执行模型到分布式时序数据库能力逐步展开的工程主线。其重点不只是单点功能实现，而是在 Rust 技术栈下完成数据库核心能力、流式订阅和分布式扩展的系统化建设。
