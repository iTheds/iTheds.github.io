## 4. 主要工作方向

基于提交主题与改动路径，可归纳为 4 条主线：

1. 基础能力建设(os_base / utilities)
- 共享内存、sleep 基础模块开发与修正
- skiplist、rbtree、错误码体系完善

2. Worker/队列/流式框架建设
- Worker 基础模型推进
- 队列从旧实现向 channel 方式演进
- Qworker 等流处理执行框架开发

3. Topic/Consumer/Subscribe 能力落地
- Topic 相关模块迭代并进入可运行状态
- Consumer/Subscribe 全流程开发、测试与命名整理
- 2024 年对订阅发布列查询继续完善(队列存储 + 查询算子过滤)

4. 服务端与协议联动改造
- 涉及 `db_common` AST/逻辑计划、`tzdb.proto`、RPC plan 转换
- 覆盖 `db_core` 多服务模块(cmngr/csvr/msvr/psvr/tq)
- 体现出跨层联动改造能力(语法 -> 协议 -> 执行/队列)
