## 记录结论

这一组记录说明，集群通信改造不是单纯把 TCP 换成 UDP：

1. 前提是先把连接复用和 `select` 多路复用模型跑通；
2. `select` 的可移植性决定了是否能统一 Win32 / Linux / ACORE_OS；
3. UDP 落地后立刻暴露帧边界、缓冲区大小、分包和重传问题；
4. 所以 IO 改造、协议改造、调试信息改造必须一起推进。
