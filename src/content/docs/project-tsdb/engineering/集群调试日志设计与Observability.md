# Cluster Debug Log Design And Observability

## 日志设计目标

原记录提出要实现“多节点并排日志视图”，用于观察 Master/Replica 同步流程。

核心目标:

- 同时展示多个节点的动作序列
- 标记动作状态码与附加信息
- 便于定位阻塞与顺序异常

## 方案要点

- 表格化日志布局(按节点列展示)
- 多对象日志打印函数抽象(含 level、code、info)
- 颜色化 pass/fail 标记

## 关键发现

- 实际阻塞关系通常是“点对点等待”，不是所有节点同时互等
- 顺序错觉可能来自“日志列顺序”与“节点 vector 顺序”不一致
- 只要等待关系形成闭环，原同步逻辑可成立
