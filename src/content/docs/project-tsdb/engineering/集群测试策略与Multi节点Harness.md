# Cluster Test Strategy And Multi Node Harness

## 测试目标

- 从单节点扩展到多节点场景验证
- 覆盖功能、异常、同步与查询正确性
- 构建可在本地复现的最小多节点测试链路

## 测试框架要点

- 基于 `tzdbGenerator` 按 toml 生成 mco 与 act 集
- 规划多级场景(Level 1/2/3)
- 查询结果至少按条数进行断言

## 关键环境问题

- 单机多节点运行受信号量与进程模型影响
- 主从分父子进程时出现阻塞问题
- 需要支持随机信号量/虚拟化以增强并行测试能力

## 异常测试方法

- 缺包场景:拦截/模拟节点不在场
- 元数据场景:替换 mco 输入构造差异
- 回调机制用于插入可观测测试点
