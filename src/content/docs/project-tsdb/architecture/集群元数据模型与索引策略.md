# Cluster Metadata Model And Index Strategy

## 目标问题

原始记录聚焦于“元数据怎么存、怎么取、怎么用”，核心诉求是支持以下查询：

- 按表名定位目标节点
- 按节点反查其承载表集合
- 在多副本/分区模式下增加分区维度查询

## 键设计演进

提出并验证了以 `库名:表名` 作为基础索引键的思路，以规避多库同名表冲突。

在分区场景下，键扩展为包含分区 ID 的组合键，以支持分区路由。

## 索引结构讨论

文档中比较了几种结构：

- 直接 `map`(稳定但扩展受限)
- 额外维护 `map<index_struct, *node>` 的多索引方式
- 按 `node/db/table` 分层 set 的结构化组织

## 最终方向(阶段结论)

- 存储结构与索引结构分离
- 以部署信息为主线维护增删改顺序
- 查询通过索引层执行，顺序扫描仅用于全量部署视图

## 增删改一致性顺序

- 添加：校验/增加存储 -> 增加部署 -> 增加索引
- 删除：删除索引 -> 删除部署 -> 删除存储
