# Cluster Deployment And Node Topology Model

## 节点与部署视图

原文给出 Master/Replica 部署示例，并围绕以下问题展开:

- 节点承载哪些库表
- 副本一致与分区副本如何共存
- 本地节点如何作为元数据决策支点

## 路由决策逻辑

- 本地节点优先判断本地是否命中目标分区
- 未命中时根据元数据定位远端节点并建立连接
- 备份/同步场景同样依赖库表定位结果

## 拓扑建模关键点

- `node list + db/table 集合` 的组合结构
- 单库部署信息与跨节点分布信息并存
- 查询条件可按需求增维(如分区 ID、角色、网络信息)

## 已发现问题

- 查询指定节点表信息时存在校验缺失 bug(需验证该表是否真在该节点)
