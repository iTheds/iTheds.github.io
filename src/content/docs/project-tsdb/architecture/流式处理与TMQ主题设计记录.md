# Stream Processing And TMQ Topic Design Notes

## 流式计算问题定义

原文指出时序场景常见链路是“清洗/预处理 -> 时序存储”，传统方案往往依赖 Kafka/Flink 等系统，带来额外开发与运维复杂度。

项目目标是将流式能力向数据库内聚，减少外部系统耦合。

## 调研重点

- TDengine 算子分布与执行流程
- 虚拟节点职责拆分
- 调试与断点策略(快速定位关键函数)
- 从概念分析过渡到接口级可实现模型

## 数据订阅与 Topic(原文要点)

- 支持基于查询语句定义 Topic
- 消费者组共享进度，不同组间进度隔离
- 依赖 ACK 机制保障至少一次消费语义
- WAL 索引支撑按消费进度快速读取
- 订阅创建分为“订阅表”与“订阅列”路径

## 节点与执行关系补充

- `dm_open_nodes` / wrapper 装载关系被明确为关键入口
- 记录了 `vnode/mnode` 结构层级和待完善方向
- 后续重点转向 mnode 与 topic 功能可运行闭环
